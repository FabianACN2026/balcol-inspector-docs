1. What is Developer Tools?

A native Salesforce utility bar component that provides four powerful developer tools directly inside Salesforce. No context switching, no external apps — everything you need is one click away in the utility bar.

Good to know

Developer Tools runs entirely within your Salesforce org. All queries and operations execute with the current user's permissions and are fully governed by Salesforce's security model.

Record Inspector

Browse any record's fields, view field-level history tracking, navigate related lists, and perform inline edits — all from a single panel.

SOQL Editor

Write queries with IntelliSense-style autocomplete, syntax highlighting, and field chips. Export results as CSV with one click.

Apex Runner

Execute anonymous Apex directly in your org with structured debug output parsing. View logs, exceptions, and system debug statements.

Data Loader

Bulk import CSV data with an interactive field mapping interface. Supports insert, update, and upsert operations with real-time progress tracking.

2. Installation

Get Developer Tools deployed and running in your org in four straightforward steps.

1

Deploy the package

Use the Salesforce CLI to deploy the component source to your target org. Run the following command from the project root:

sf project deploy start \
  --source-dir force-app/main/default/classes \
  --source-dir force-app/main/default/lwc/devTools \
  --target-org yourOrgAlias \
  --wait 10Terminal
Note

Do not include FlexiPage or CustomApplication metadata in the deploy. These have pre-existing XML conflicts and should be configured manually in your org.

2

Assign the permission set

Grant access by assigning the ABN_Dev_Tools_Admin permission set to each user who needs the tools. You can do this from Setup or via CLI:

sf org assign permset \
  --name ABN_Dev_Tools_Admin \
  --target-org yourOrgAliasTerminal
Permission Set Access Level
Dev_Tools_Access Read-only access (inspect records, run queries)
Dev_Tools_Power Full access (includes Apex Runner, Data Loader, inline edits)
3

Add the component to a utility bar

Navigate to Setup → App Manager, edit the Lightning App of your choice, and add devTools to the Utility Items section. Set the panel height to at least 480px for optimal layout.

Tip

We recommend adding Developer Tools to an app that your developers use daily, such as your custom admin app or the default Sales app.

4

Refresh and launch

Hard-refresh your browser (Ctrl + Shift + R), then look for the Developer Tools icon in the utility bar at the bottom of the page. Click it to open.

Done

You are now ready to use Developer Tools. Continue reading for a walkthrough of the SOQL Editor and Record Inspector.

3. Your First Query

The SOQL Editor lets you write, run, and export Salesforce queries with zero setup. Here is a quick walkthrough.

Write the query

Click on the SOQL Editor tab in Developer Tools. You will see a blank editor. Type the following query:

SELECT Id, Name, Industry
FROM Account
LIMIT 10SOQL

Autocomplete

As you type, the editor offers IntelliSense-style autocomplete suggestions:

  • Object names appear after the FROM keyword
  • Field names appear after SELECT, WHERE, and ORDER BY
  • Relationship fields are traversed up to 4 levels deep (e.g., Account.Owner.Name)

Accept a suggestion by pressing Tab or Enter. Dismiss suggestions with Esc.

Field chips

When typing a field name, interactive chips appear below the editor showing matching fields with their data types. This gives you at-a-glance visibility into the schema without leaving the editor.

Run and view results

Press Ctrl + Enter to execute the query. Results appear in a sortable, scrollable table below the editor. The query uses cursor-based pagination, so there is no 2,000-row limit — large result sets are streamed automatically in batches of 200.

Export

Click the Export CSV button above the results table to download all rows as a comma-separated file. The export includes all columns visible in the result table.

Multiple tabs

You can open multiple query tabs to work on different queries simultaneously. Click the + button next to the existing tabs to create a new editor tab. Each tab maintains its own query, results, and history.

4. Inspecting a Record

The Record Inspector gives you a complete view of any Salesforce record — every field, its value, and its metadata — without navigating away from your current page.

Open a record

Switch to the Record Inspector tab and paste a 15- or 18-character Record ID into the search bar. Developer Tools automatically detects the object type and loads all field data.

Tip

You can also type an object name (e.g., "Account") to browse object metadata, including field definitions and child relationships.

Browse fields

All fields are displayed in a filterable, scrollable list. Each field row shows:

Column Description
Label The field's display label (e.g., "Account Name")
API Name The field's developer name (e.g., Name)
Value The current value on the record
Type The Salesforce data type (String, DateTime, Picklist, etc.)

Use the search filter at the top to narrow the list by label or API name.

Inline editing

Click on any editable field value to modify it directly. After making your change, press the Save button to commit the update. Fields that are not editable (formula fields, system fields) are shown as read-only.

Caution

Inline edits are saved immediately to the database. There is no undo. Make sure you are editing the correct record before saving.

Related lists

Below the field list, you can browse the record's child relationships. Each relationship shows whether it has records or is empty, and you can expand it to view related data directly.
